### 2. INTRODUCTION (Pendahuluan)

#### 2.1 Permasalahan (HEADING_2)

Banyak sistem elektronik dan industri kecil yang memerlukan manajemen termal aktif untuk menjaga suhu operasi komponen dalam rentang aman. Sistem kontrol suhu komersial (PLC, PID controller dedicated) memiliki biaya tinggi dan kompleksitas yang berlebihan untuk aplikasi skala kecil. Solusi berbasis ESP32 dengan sensor digital BMP280 dan aktuator kipas DC yang dikontrol PWM dapat menjadi alternatif biaya rendah yang tetap memberikan akurasi memadai, kemampuan IoT, dan fleksibilitas kontrol.

#### 2.2 Solusi yang Diharapkan (HEADING_2)

Sistem yang dapat:
- Membaca suhu lingkungan secara digital melalui sensor BMP280 via I2C (akurasi ±1.0°C)
- Mengontrol kecepatan kipas 12V DC secara proporsional terhadap error suhu dengan kurva proportional gain
- Beroperasi dalam dua mode: AUTO (setpoint-based dengan hysteresis) dan MANUAL (PWM langsung)
- Menyediakan dashboard web real-time dan kontrol dari browser
- Menyimpan data log suhu ke LittleFS dan mengekspor dalam format CSV dan GNUplot

#### 2.3 Tujuan (HEADING_2)

1. Implementasi pembacaan suhu BMP280 via I2C pada ESP-IDF dengan kompensasi suhu internal sensor
2. Implementasi kontrol PWM L298N dengan proportional gain, ramp protection, dan hysteresis
3. Implementasi web server ESP32 dengan REST API dan dashboard Chart.js real-time
4. Implementasi data logging ke LittleFS dengan ring buffer + flash persistence
5. Implementasi dual-mode control (AUTO/MANUAL) melalui web dan tombol fisik
6. Verifikasi sistem melalui pengukuran dan analisis data

#### 2.4 Batasan Permasalahan (HEADING_2)

- **Sensor**: BMP280 (I2C, GPIO21=SDA, GPIO22=SCL, alamat 0x76, clock 100kHz)
- **Aktuator**: Kipas 12V DC via L298N (PWM 5kHz, 8-bit, GPIO16/17/18)
- **Kontrol**: Button GPIO4 (mode) dan GPIO5 (value), debounce 50ms, long-press 500ms
- **Catu daya**: 3S Li-ion 11.1V → BMS → L298N VS → Regulator 5V → ESP32
- **Logging**: Ring buffer 100 entri RAM, flush ke LittleFS tiap 10s, rolling max 512 entri
- **Jaringan**: WiFi STA (prioritas, SSID/password via menuconfig) → fallback AP ("ThermalCtrl-AP")
- **Safety**: Override 100% pada >60°C, shutdown kipas pada >75°C

#### 2.5 Konsep Pengukuran Masalah (HEADING_2)

Parameter yang diukur: suhu lingkungan (°C) dari BMP280, kecepatan kipas (PWM 0-255 setara 0-100%), mode operasi (AUTO/MANUAL), setpoint suhu, dan tegangan baterai. Data direkam dengan timestamp millis() menggunakan ESP timer untuk analisis respon termal sistem terhadap perubahan suhu.

Penjelasan pemilihan BMP280:
- **Interface digital I2C**: Lebih tahan noise dibanding analog — tidak terpengaruh panjang kabel atau interferensi elektromagnetik dari motor/L298N
- **Data tekanan tambahan**: Dapat digunakan untuk kalibrasi ketinggian atau analisis lanjutan
- **Resolusi 20-bit**: Suhu dibaca langsung dalam format digital terkompensasi, tidak perlu kalibrasi ADC manual
- **Daya sangat rendah**: Mode standby 1.8µA cocok untuk aplikasi battery-powered

#### 3.4 I2C Communication Protocol (HEADING_2)

BMP280 diakses melalui bus I2C dengan konfigurasi:
- Frekuensi clock: 100kHz (standard mode)
- Alamat slave: 0x76 (ketika pin SDO terhubung ke GND)
- Pull-up resistor: 4.7kΩ pada SDA dan SCL (internal ESP32 pull-up tidak mencukupi untuk I2C)

**Register map BMP280:**
- 0xD0: Chip ID (diharapkan 0x58)
- 0x88-0x8F: Kalibrasi NVM (dig_T1, dig_T2, dig_T3 — 24 byte)
- 0xF4: CTRL_MEAS — konfigurasi oversampling dan mode
- 0xF5: CONFIG — konfigurasi rate dan filter
- 0xF7-0xFC: Data tekanan dan suhu (6 byte)

**Algoritma pembacaan suhu:**
1. Write register address 0xF7 (MSB data suhu)
2. Read 3 bytes (MSB + LSB + XLSB) via I2C repeated start
3. Gabungkan: `adc_T = (msb << 12) | (lsb << 4) | (xlsb >> 4)`
4. Kompensasi suhu menggunakan koefisien kalibrasi dari NVM:
   ```
   var1 = (((adc_T>>3) - (dig_T1<<1)) * dig_T2) >> 11
   var2 = ((((adc_T>>4) - dig_T1) * ((adc_T>>4) - dig_T1) >> 12) * dig_T3) >> 14
   t_fine = var1 + var2
   temp_C = (t_fine * 5 + 128) >> 8  // dalam 0.01°C, bagi 100
   ```

#### 3.5 Interkoneksi Sensor ke Mikrokontroler (HEADING_2)

**BMP280:**
- VCC → ESP32 3.3V (output 3.3V pin)
- SDA → GPIO21 (via 4.7kΩ pull-up ke 3.3V)
- SCL → GPIO22 (via 4.7kΩ pull-up ke 3.3V)
- GND → GND common
- CSB → 3.3V (non-aktif SPI mode)
- SDO → GND (I2C address = 0x76)

**L298N + Fan:**
- VCC (5V logic) → ESP32 5V (VIN pin, dari regulator L298N)
- VS (12V motor supply) → Baterai 3S Li-ion (+)
- GND → GND common
- ENA → GPIO16 (PWM input)
- IN1 → GPIO17 (HIGH — selalu forward)
- IN2 → GPIO18 (LOW)
- OUT1/OUT2 → Terminal kipas 12V DC

**Catu Daya:**
- 3S Li-ion (11.1V nominal) → BMS → L298N VS
- L298N internal 5V regulator → ESP32 VIN
- Kapasitor bulk 470µF pada rail 5V untuk mencegah brownout ESP32
- Ground semua perangkat disatukan (common ground)

#### 3.7 Algoritma Kontrol dan Flowchart (HEADING_2)

**Main system flowchart:**

```
app_main()
  ├── Init: sensor (I2C), actuator (PWM), input (GPIO), storage (SPIFFS)
  ├── Init: WiFi STA (10s timeout → fallback AP), HTTP server
  │
  ├── ESP_TIMER (50ms) → read_temp()
  │     └── i2c_read(BMP280) → temp_compensate()
  │
  ├── ESP_TIMER (100ms) → scan_buttons()
  │     └── debounce() → short_press() / long_press()
  │
  ├── ESP_TIMER (200ms) → control_fan()
  │     ├── if mode == AUTO:
  │     │     error = setpoint - temp_c
  │     │     if |error| < 0.5°C: hold
  │     │     elif error <= 0: pwm = 0
  │     │     else: pwm = error × 25.5
  │     ├── if mode == MANUAL: pwm = manual_setpoint
  │     ├── Safety: temp > 60°C → pwm=255; temp > 75°C → pwm=0
  │     ├── Ramp: max delta = 40/step (mechanical shock protection)
  │     └── ledc_set_duty() → log_entry_push()
  │
  └── while(1):
        ├── network_poll() — reconnect if STA disconnected
        └── flush ring buffer → LittleFS every 10s
```

**Button state machine:**
- GPIO4 falling edge → toggle AUTO/MANUAL (delta=0 callback)
- GPIO5 falling edge → start timer, mark press time
- GPIO5 held ≥500ms → coarse adjust ±10 with long_press flag
- GPIO5 rising edge <500ms → fine adjust ±1 (short press)

**Fan control law (AUTO mode):**
- P gain = 25.5 (full PWM 255 pada error 10°C)
- Hysteresis deadband: ±0.5°C (mencegah osilasi di sekitar setpoint)
- Ramp rate: maksimum 40 step per update cycle (200ms → 200 step/s, soft start ~1.25s)
- Minimum duty: 80/255 (~31%) — diperlukan untuk mengatasi inersia startup kipas 12V

**Safety logic:**
- Suhu > 60°C: override 100% PWM (terlepas dari mode)
- Suhu > 75°C: matikan kipas, log error (critical shutdown)

### 6. MULTI-REALISTIC CONSTRAINT ANALYSIS (HEADING_1)

Table: Kendala | Mitigasi

| Constraint | Mitigasi |
|---|---|
| Power brownout (drop tegangan saat fan start) | Kapasitor bulk 470µF pada rail 5V L298N. ESP32 brownout detector. State tersimpan di RTC memory. |
| Fan inrush current (arus start tinggi) | Soft-start ramp 200 step/s. L298N rated 2A continuous (4× margin dari 500mA fan). |
| Flyback voltage (inductive spike dari motor) | L298N memiliki clamping diodes built-in pada output. |
| I2C bus noise (interferensi dari motor) | I2C digital immune terhadap noise elektromagnetik. Pull-up 4.7kΩ. Clock 100kHz (slow mode). |
| WiFi disconnect | Reconnect timer 30s. Kontrol button tetap berfungsi offline. Banner "Offline" di web. |
| Data loss (crash/restart) | Dual buffer: RAM ring + LittleFS persist. Maks 10s data loss. |
| Fan mechanical wear | Ramp 200 step/s. Minimum duty ~31%. Hysteresis ±0.5°C cegah osilasi. |
| Thermal runaway (kegagalan sensor/kontrol) | 60°C → 100% fan override. 75°C → shutdown kipas. Dua level safety independen. |
| Real-time requirement | ESP timer callbacks (tidak ada delay()). Prioritas: 50ms temp → 100ms button → 200ms fan. Main loop yield tiap 10ms. |
| Li-ion battery safety | 3S BMS (protection: overcharge, overdischarge, short circuit). Monitoring tegangan via ADC GPIO35. Low-bat warning via web. |
